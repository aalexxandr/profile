# Чеклисты для ИИ-агентов

## Перед кодом

- Прочитай `AGENTS.md`.
- Найди похожие файлы через `rg`/`rg --files`.
- Если задача касается структуры, слоев, слайсов, импортов или размещения assets, используй skill `feature-sliced-design`.
- Для Next.js прочитай релевантный файл в `node_modules/next/dist/docs/`.
- Для внешних библиотек получи docs через Context7.
- Проверь, не нужен ли Figma, Playwright, Chrome DevTools или Next DevTools MCP.

## Во время реализации

- Держи страницы `src/app` тонкими.
- Делить крупные UI-блоки на компоненты.
- Выносить повторяемую и нетривиальную логику в hooks или helpers.
- FSD-границы и импорты проверять по skill `feature-sliced-design`.
- Не добавлять `"use client"` выше минимально нужной границы.
- Добавлять пользовательские тексты в оба словаря.

## Перед финалом

- Запусти релевантные проверки:

```bash
bun run lint
bun run format:check
bun run build
```

- Для архитектурных изменений попробуй:

```bash
bun run fsd:check
```

- Если проверку не удалось запустить, явно напиши почему.
- В финальном ответе перечисли измененные области и результат проверок.

## UI-задачи

- Проверить desktop и mobile viewport.
- Проверить, что текст не обрезается и не перекрывает соседние элементы.
- Проверить focus/hover/active состояния для интерактивных элементов.
- Проверить console/runtime errors через browser tooling, если запускался dev-сервер.

## Архитектурные изменения

- Используй skill `feature-sliced-design` как источник правил для структуры, слоев, слайсов, public API, импортов и assets.
- `bun run fsd:check` запускается отдельно от общего `check`, пока не устранен `EMFILE` blocker.
